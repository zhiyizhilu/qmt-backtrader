# 小市值策略 — 降低回撤、提升收益

## 一、策略背景

小市值策略的核心逻辑是：小盘股由于机构覆盖少、信息不对称大、流动性溢价低，长期存在超额收益（Size Factor）。A股历史上小市值因子表现极为强势，年化收益远超大盘。

但问题同样突出：

- **回撤大**：小盘股在熊市中跌幅远超大盘，2015年股灾、2018年去杠杆、2022年成长股杀估值，小市值策略回撤动辄40%~60%
- **收益不稳定**：小盘股风格切换剧烈，牛市暴涨、熊市暴跌，收益分布严重右偏
- **流动性风险**：小盘股成交低迷，冲击成本高，实盘滑点大
- **壳价值衰减**：注册制推行后，壳资源价值大幅缩水，小市值因子长期溢价在收窄

## 二、策略逻辑

### 选股流程

```
股票池 → 基本面过滤 → 市值计算 → [市值上限过滤] → 动量过滤 → 行业分散选股 → 取前N只
```

### 1. 基本面质量过滤 — 剔除"垃圾小盘"

小市值策略最大的坑，是选到了基本面恶化的"垃圾股"。纯按市值排序选股，容易选到亏损、高负债、现金流断裂的公司，这些股票在熊市中跌幅最深。

**过滤条件：**

| 条件 | 数据来源 | 字段 | 阈值 |
|------|---------|------|------|
| ROE > 0 | Pershareindex | `du_return_on_equity` | `min_roe=0.0` |
| 营收增速 > 0 | Pershareindex | `inc_revenue_rate` | `min_revenue_growth=0.0` |
| 经营现金流 > 0 | Pershareindex | `s_fa_ocfps` | `min_operate_cashflow=0.0` |
| 资产负债率 < 60% | Balance | `total_liabilities / total_assets` | `max_debt_ratio=0.6` |

**容错机制：** 当某个字段数据缺失时，跳过该条检查而非直接排除股票；仅当所有基本面字段（ROE、营收增速、经营现金流）全部缺失时才排除。这样避免因数据覆盖不全导致大量股票被误杀。

**降级机制：** 当所有股票的财务数据全部缺失时，若 `skip_fundamental_if_missing=True`，则跳过基本面过滤，返回整个股票池。

### 2. 市值计算

市值 = 总股本 × 当前股价，其中总股本通过以下方式获取：

- **优先方案**：总股本 = 所有者权益合计（`Balance.total_equity`） / 每股净资产（`Pershareindex.s_fa_bps`）
- **回退方案**：当每股净资产不可用时，使用所有者权益合计作为市值代理

无法获取价格或股本数据的股票会被跳过，并在日志中统计数量。

### 3. 市值上限过滤（可选）

通过 `max_market_cap` 参数设置市值上限（单位：亿），剔除市值过大的股票，确保策略聚焦于真正的小盘股。默认 `None` 表示不限制。

### 4. 短期动量过滤 — 避免接飞刀

纯小市值策略容易在下跌趋势中不断买入"便宜"的股票，结果越买越跌。

**过滤逻辑：**
- 取过去 `momentum_period`（默认20）个交易日的收盘价
- 动量 = (最新价 - N日前价) / N日前价
- 动量 > `min_momentum`（默认0）的股票通过过滤

**容错机制：** 动量过滤为空时不强制过滤，保留全部市值有效股票，避免在震荡市中过度筛选导致无股可选。

### 5. 行业分散选股 — 避免集中暴露

小盘股往往集中在特定行业（如TMT、机械、化工），行业集中会导致：
- 行业周期下行时，组合整体大幅回撤
- 某些行业的小盘股质量普遍较差

**做法：**
- 申万一级行业分类，每个行业选市值最小的1只
- 行业中性化处理：每个行业选市值最小的，而非全市场选市值最小的
- 无行业分类的股票归入"未知"行业，同样选市值最小的1只

### 6. 最终排序与持仓

- 行业代表按市值升序排列，取前 `max_stocks`（默认10）只
- 等权重持仓，仓位比例 `position_ratio`（默认95%），5%留作现金缓冲
- 月度调仓（`rebalance_freq='monthly'`）

## 三、策略参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `rebalance_freq` | `'monthly'` | 调仓频率 |
| `max_stocks` | `10` | 最大持仓股票数 |
| `position_ratio` | `0.95` | 仓位比例，5%留作现金缓冲 |
| `stock_pool` | `None` | 股票池，None则从财务数据缓存获取 |
| `min_roe` | `0.0` | ROE最低阈值 |
| `min_revenue_growth` | `0.0` | 营收增速最低阈值 |
| `min_operate_cashflow` | `0.0` | 经营现金流最低阈值 |
| `max_debt_ratio` | `0.6` | 资产负债率上限 |
| `momentum_period` | `20` | 动量回看天数 |
| `min_momentum` | `0.0` | 动量最低阈值 |
| `max_market_cap` | `None` | 市值上限（亿元），None表示不限制 |
| `skip_fundamental_if_missing` | `True` | 所有股票财务数据缺失时跳过基本面过滤 |

## 四、策略实现

```python
from collections import defaultdict
from typing import Dict, List, Optional
from core.stock_selection import StockSelectionStrategy
from strategies import register_strategy


@register_strategy('small_cap', default_kwargs={'max_stocks': 10},
                   backtest_config={'cash': 1000000, 'commission': 0.0001,
                                    'start_date': '2016-01-01', 'end_date': '2026-04-17'})
class SmallCapStrategy(StockSelectionStrategy):
    """小市值优化策略 - 基本面过滤+行业分散+动量确认

    选股逻辑：
    1. 基本面过滤：ROE > 0、营收增速 > 0、经营现金流 > 0、资产负债率 < 阈值
    2. 行业分散：申万一级行业各选市值最小的1只
    3. 动量确认：近N日涨幅 > 0
    4. 按市值升序排列，取前N只
    5. 等权重持仓，月度调仓

    降低回撤机制：
    - 基本面过滤：排除"垃圾小盘"，剔除亏损、高负债、现金流断裂的公司
    - 行业分散：避免重仓单一周期行业
    - 动量确认：避免在下跌趋势中接飞刀
    - 建议配合 RiskController 使用（单股止损8%、组合最大回撤15%）
    """

    params = (
        ('rebalance_freq', 'monthly'),
        ('max_stocks', 10),
        ('position_ratio', 0.95),
        ('stock_pool', None),
        ('min_roe', 0.0),
        ('min_revenue_growth', 0.0),
        ('min_operate_cashflow', 0.0),
        ('max_debt_ratio', 0.6),
        ('momentum_period', 20),
        ('min_momentum', 0.0),
        ('max_market_cap', None),
        ('skip_fundamental_if_missing', True),
    )
```

**风险提示：**
1. 注册制全面推行后，小市值因子长期溢价可能持续收窄
2. 小盘股流动性差，实盘冲击成本远高于回测
3. 任何历史回测都不能保证未来收益
4. 当前策略未启用 RiskController 风控，实盘使用建议配合止损和回撤控制
