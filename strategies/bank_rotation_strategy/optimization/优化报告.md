## 十、优化洞察与结论

### 核心发现

**降低换仓阈值**是唯一有效的优化方向。原始策略的换仓条件与建仓条件使用相同阈值（0.005），导致持仓时换仓不够灵敏。将换仓阈值降至0.003后，策略能更积极地从相对偏贵的银行股切换到相对便宜的银行股，捕获更多均值回归收益。

### 无效优化分析

| 优化方向 | 失败原因 |
|----------|----------|
| 提高换仓阈值 | 进一步降低换仓灵敏度，错失更多机会 |
| 波动率过滤 | 银行股本身波动率低，过滤无效果 |
| 最小持仓期 | 限制了策略的灵活性，降低换仓效率 |
| 开盘/收盘过滤 | 过滤掉的时段存在有效信号 |
| MA趋势过滤 | 银行股趋势性弱，过滤导致长期空仓 |
| 自适应阈值 | 增加了不必要的复杂性，降低了信号质量 |
| 比率确认 | 延迟了交易时机，均值回归信号转瞬即逝 |
| 当日回撤限制 | 银行股日内波动有限，限制反而阻碍正常换仓 |
| 扩展标的 | 新增标的与原四大行比率计算逻辑不同，无增量信息 |

### 策略本质

银行轮动策略的核心竞争力在于**高频捕捉四大行之间的微小定价偏差**。任何降低交易频率或增加交易阻力的优化都会损害策略表现。唯一有效的改进方向是**让换仓更灵敏**，即降低换仓阈值。
