# 银行轮动策略优化报告 - 基于Tick级买卖价差滑点

> 优化日期：2026-05-28
> 策略路径：`strategies_for_vip/bank_rotation_strategy/`
> 回测模式：`--period 1m --ai-mode`
> 核心变更：**将Tick级买卖价差侵蚀比例固化为回测滑点参数**

---

## 一、优化背景

### 1.1 问题发现

原策略使用 `switch_threshold=0.003` 作为换仓阈值，回测时使用收盘价成交（无滑点），年化收益极高。但实际交易中，买入需按卖一价(ask1)成交，卖出需按买一价(bid1)成交，存在买卖价差侵蚀。

### 1.2 核心问题

| 指标 | 数值 |
|------|------|
| 买卖价差导致的单向价格侵蚀 | 0.14%（非零价差均值） |
| 双向价差均值 | 0.139% |
| 单次换仓总成本（双向滑点+双向佣金） | 0.32% |
| threshold=0.003时成本占比 | 53.3% |
| threshold=0.003时净利润空间 | 仅0.28% |

**结论：0.003不是好的交易阈值，超过一半的换仓利润被价差吞噬。**

---

## 二、Tick级价格侵蚀精确计算

### 2.1 计算方法

使用2026年5月的Tick级数据（4只银行股），计算：

- **买入侵蚀** = (ask1 - lastPrice) / lastPrice
- **卖出侵蚀** = (lastPrice - bid1) / lastPrice
- **双向价差** = (ask1 - bid1) / lastPrice

### 2.2 计算结果

| 指标 | 均值 | 中位数 | P75 | P90 |
|------|------|--------|-----|-----|
| 单向价格侵蚀 | 0.067% | 0.000% | 0.098% | 0.196% |
| 非零价差单向侵蚀 | ~0.14% | - | - | - |
| 双向价差 | 0.139% | 0.000% | 0.196% | 0.392% |

> 注：中位数为0是因为大量tick的ask1=lastPrice（无价差），但交易发生时市场在波动，价差几乎一定存在。因此取**非零价差均值0.14%**作为实际滑点。

### 2.3 滑点固化

将 `slippage=0.0014` 固化到策略的 `backtest_config` 中：

```python
backtest_config={'cash': 200000, 'commission': 0.0002,
                 'slippage': 0.0014,  # 基于Tick级买卖价差计算
                 ...}
```

框架逻辑：
- 买入成交价 = price × (1 + 0.0014)
- 卖出成交价 = price × (1 - 0.0014)

## 四、其他优化项测试

在最优参数 (switch_threshold=0.008, spread_threshold=0.003) 基础上测试：

| 优化项 | 夏普 | 收益 | 结论 |
|--------|------|------|------|
| 波动率过滤(0.02-0.04) | 0.47 | 18.35% | 无变化（银行股波动率低，不触发） |
| 每日最大换仓2-4次 | 0.47 | 18.35% | 无变化（本身换仓不频繁） |
| 扩展标的(+交通银行) | 0.47 | 18.35% | 无变化 |
| 自适应阈值(0.5/1.0) | 0.47 | 18.35% | 无变化 |

**结论：在含滑点条件下，额外优化项均无正向贡献，策略已足够简洁。**

## 七、风险提示

1. **年化收益低**：含滑点后年化仅~3-4%，跑输大部分固收产品
2. **样本外衰减明显**：验证集夏普仅为测试集的51%
3. **2025-2026表现差**：银行股波动率下降导致轮动机会减少
4. **策略容量有限**：四大行流动性虽好，但轮动频率低
5. **滑点假设保守**：实际大单交易可能面临更大价差
